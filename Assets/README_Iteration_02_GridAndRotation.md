# README — Iteration 02: Grid + Pieces + Rotation (World 1 core)

## 1. Что было добавлено
Игровая сетка с auto-fit (квадратное поле, центрируется, размер клетки подбирается под экран), отрисовка элементов пути (Straight / Corner / Triple / Cross) спрайтами, сгенерированными кодом, и вращение элемента по тапу на 90° с анимацией, звуком и вибрацией. Это играбельное ядро Мира 1 (только вращение). Создаются 2 тестовых уровня (4x4 и 5x5).

## 2. Что изменилось с прошлой итерации
Игровая сцена больше не пустая заглушка — на ней появляется поле. Кнопка BACK и всё из итерации 1 на месте. Менеджеры не трогались.

## 3. Editor скрипты: что запустить
ВАЖНО: итерация 1 уже должна быть установлена (Tools → Loop Puzzle → Setup Iteration 1 хотя бы раз запущен, сцены существуют).

1. Скопируй папку `Assets/LoopPuzzle/` из архива поверх своего проекта (новые файлы добавятся).
2. Дождись компиляции без ошибок.
3. В меню Unity: **Tools → Loop Puzzle → Setup Iteration 2**

Скрипт:
- создаст 2 ассета уровней в `Assets/LoopPuzzle/Levels/` (Level_01, Level_02)
- добавит на сцену Game объекты GridManager и PieceInput, прокинет ссылки
- назначит Level_01 текущим уровнем
- сохранит сцену

## 4. Как тестировать
1. Открой сцену `Assets/LoopPuzzle/Scenes/Game.unity`.
2. Нажми **Play**.
3. Тапай по элементам пути — каждый поворачивается на 90°.
4. Поверни все 4 угла так, чтобы они образовали замкнутый квадрат (визуально линии соединяются без открытых концов). Победы пока нет — это итерация 3, просто проверь что вращение работает и соединения читаются.
5. Чтобы проверить 5x5: выбери объект GridManager на сцене → в поле Current Level поставь Level_02 → Play.

## 5. Ожидаемый результат
- Поле появляется по центру, клетки — полупрозрачные квадраты, на стартовой клетке оранжевое кольцо-маркер.
- Элементы пути нарисованы голубым: видно, в какие стороны смотрят коннекторы.
- Тап по элементу → плавный поворот с лёгким «отскоком» (Ease.OutBack) + punch-масштаб + звук + вибрация на устройстве.
- Поле 4x4 даёт крупные клетки, 5x5 — мельче (auto-fit работает).
- Стартовый маркер остаётся на клетке при вращении элемента (он маркирует клетку, не элемент).

## 6. Известные ограничения текущей итерации
- Проверки победы пока НЕТ (будет в итерации 3) — loop собирается «на глаз».
- Перемещения элементов пока НЕТ (drag будет в итерации 5) — элементы крутятся на месте.
- HUD (номер уровня, счётчик ходов) пока нет — итерация 3.
- Triple/Cross нарисованы и работают логически, но в тестовых уровнях пока не используются.
- Спрайты — кодовые плейсхолдеры, потом заменятся ассетами.
- Переключение уровня — пока вручную через поле в GridManager (экран выбора — итерация 6).

## 7. Что будет в следующей итерации
Проверка замкнутого loop, попапы победы/рестарта, HUD со счётчиком ходов — замкнём базовый game-loop.
