# Loop Puzzle — Итерация 15: Color Loops

Самая «вкусная» механика на фундаменте итерации 11, плюс 3 демо-уровня (35–37).

## Механика

Элементы бывают **разных цветов** (красный, синий, зелёный, жёлтый). Нужно собрать **отдельную замкнутую петлю под каждый цвет** — цвета нельзя смешивать: элементы разных цветов не соединяются друг с другом.

Условие победы (Color Loops):
- Каждый цвет, присутствующий на поле, образует свою замкнутую петлю.
- Все элементы задействованы (нет «висящих» концов).
- Соединения только между элементами одного цвета (либо с нейтральным).

Это добавляет целое измерение: игрок должен мыслить не «одна петля», а «несколько независимых петель», сортируя элементы по цвету.

## Демо-уровни (35–37)

- **35** — два цвета: красная и синяя петли 2×2 рядом (5×2).
- **36** — два цвета: красная и синяя петли друг над другом (2×5).
- **37** — три цвета: красная, синяя и зелёная петли 2×2 на поле 5×5.

В демо элементы каждого цвета уже на своих местах, но повёрнуты — игрок поворачивает их, замыкая каждую цветную петлю. Если повернуть угловой элемент так, что он «потянулся» бы к чужому цвету — это не считается соединением, петля не закроется.

## Как обеспечена корректность

Усилил проверку победы (`ValidateColorLoops`) до строгой:
- Каждый присутствующий цвет образует замкнутую петлю.
- **Нет ни одного открытого конца** (все элементы в петлях).
- Число элементов в петлях равно общему числу элементов (ничего не висит).
- Соединение между разными цветами не засчитывается (через `ColorsCompatible`).

Каждый демо-уровень построен программно и проверен:
- Решение: каждый элемент в замкнутой петле своего цвета, без открытых концов.
- Старт не решён (элементы перемешаны — петли разорваны даже без учёта цвета).
- Реконструкция (поворот в решение) выигрывает.
- Логика идентична в Python-движке (`is_color_win`) и в игровом валидаторе.
- Проверено, что игровой валидатор не считает старт пре-решённым.
- Все 34 предыдущих уровня (мосты, ломающиеся клетки, стрелки) по-прежнему валидны.

## Установка

1. Нужны выполненные Setup итераций 1–14.
2. Скопируй `Assets/LoopPuzzle/` поверх итерации 14.
3. Запусти `Tools → Loop Puzzle → Setup Iteration 15`.

Setup переимпортирует все 37 уровней (с полями цвета и режима Color Loops), провалидирует, пересоберёт базу. Появятся уровни 35–37.

## Как проверить

- `Tools → Loop Puzzle → Validate All Levels` — все 37 уровней валидны.
- `Unlock All Levels`, открой **35/36/37** — увидишь элементы двух-трёх цветов. Замкни отдельную петлю каждого цвета. Победа засчитается только когда ВСЕ цвета образуют свои петли.

## Заметки

- Цвета элементов берутся из `PieceColor` (красный/синий/зелёный/жёлтый), окраска через `ApplyColor` — контрастные, хорошо различимые.
- Анимация потока на победе проходит по петле, в которой находится стартовый маркер (одна из цветных петель). Остальные петли в этот момент не анимируются — на корректность победы это не влияет. Множественную анимацию всех петель можно добавить отдельной полировкой.
- В демо элементы на местах (только поворот) — это чисто демонстрирует механику цветных петель. Уровни со смешиванием и перетаскиванием по цвету логично сделать в финальном ребалансе (итерация 17).
