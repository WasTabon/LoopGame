# README — Iteration 05: Free Drag (World 2 core)

## 1. Что было добавлено
Перемещение элементов перетаскиванием (free drag) — ядро Мира 2. Зажимаешь Movable-элемент, тащишь в любую пустую клетку, отпускаешь — элемент встаёт (snap); если клетка занята/невалидна/вне поля — возврат на место. Во время перетаскивания все валидные пустые клетки подсвечиваются, клетка под пальцем — ярче. Создаются 2 уровня Мира 2 с изначально смещёнными элементами.

## 2. Что изменилось с прошлой итерации
- `PieceInput` теперь полноценная машина состояний: короткий тап = поворот (как раньше), движение за порог = drag. Без конфликтов между тапом и перетаскиванием.
- `GridManager` получил WorldToCell-преобразование, перемещение элемента между клетками и обход всех клеток.
- `Cell` умеет подсвечиваться как валидная цель / цель под пальцем.
- `PathPiece` получил анимации подъёма (Lift), снапа (SnapTo) и возврата (ReturnTo).
- `SoundManager` получил звуки pickup/drop.
- Перемещение считается ходом и запускает проверку победы (как поворот).
- Всё из итераций 1–4 на месте.

## 3. Editor скрипты: что запустить
ВАЖНО: итерации 1–4 уже должны быть установлены.

1. Скопируй папку `Assets/LoopPuzzle/` из архива поверх своего проекта.
2. Дождись компиляции без ошибок.
3. В меню Unity: **Tools → Loop Puzzle → Setup Iteration 5**

Скрипт создаст 2 ассета уровней Мира 2 (Level_03 — 4x4, Level_04 — 5x5) в `Assets/LoopPuzzle/Levels/`. В этих уровнях часть элементов лежит НЕ на своих местах — их надо перетащить в пустые клетки, потом повернуть.

## 4. Как тестировать
1. Открой сцену `Assets/LoopPuzzle/Scenes/Game.unity`.
2. Выбери объект **GridManager** → в поле **Current Level** поставь **Level_03**.
3. Нажми **Play**.
4. В Level_03 один угол лежит в дальнем углу поля. Зажми его и перетащи в пустую клетку центрального квадрата → отпусти (встанет). Потом поверни все 4 угла правильно → победа.
5. Проверь возврат: начни тащить элемент и брось его на занятую клетку или мимо поля — он вернётся на место со звуком отказа.
6. Проверь, что обычный короткий тап по-прежнему вращает.
7. Повтори на Level_04 (5x5) — там смещены два элемента.
8. Мир 1 (Level_01/02) работает как раньше — там двигать ничего не нужно, только крутить.

## 5. Ожидаемый результат
- При захвате элемент слегка увеличивается и поднимается над полем (поверх остальных).
- Все пустые валидные клетки подсвечиваются голубым; клетка под пальцем — оранжевым.
- Бросок в валидную клетку: snap с «отскоком», звук drop, ход засчитан, запускается проверка победы.
- Бросок в невалидную: плавный возврат на исходную клетку, звук отказа, ход НЕ засчитан.
- Fixed-элементы (если появятся в уровнях) нельзя ни двигать, ни крутить.

## 6. Известные ограничения текущей итерации
- Звёзды в попапе всё ещё плейсхолдер — итерация 6.
- NEXT по-прежнему перезагружает текущий уровень — итерация 6.
- Переключение уровня — вручную через GridManager (экран выбора — итерация 6).
- Obstacle/Fixed-клетки поддержаны в коде, но в тестовых уровнях пока почти не используются (контент Миров 2–3 наполним в итерации 6).
- Drag реализован для одного пальца (мультитач для перетаскивания нескольких элементов не предусмотрен — по ТЗ не требуется).
