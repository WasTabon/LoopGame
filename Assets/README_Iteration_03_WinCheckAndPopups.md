# README — Iteration 03: Win Check + Popups + HUD

## 1. Что было добавлено
Проверка победы (базовое условие из ТЗ — существует хотя бы один замкнутый loop без открытых концов), всплывающий попап победы с анимацией, sticky-HUD сверху (номер уровня + счётчик ходов с анимацией), кнопки Restart и Next. Проверка запускается автоматически после каждого поворота.

## 2. Что изменилось с прошлой итерации
- `GridManager` теперь запоминает стартовую клетку и умеет не строить уровень сам (этим управляет LevelController).
- `PathPiece.Rotate()` принимает callback и вызывает проверку после докрутки.
- `PieceInput` сообщает о ходах LevelController и блокирует ввод во время победной анимации.
- `SoundManager` получил победный звук (арпеджио).
- Всё из итераций 1–2 на месте.

## 3. Editor скрипты: что запустить
ВАЖНО: итерации 1 и 2 уже должны быть установлены (сцены и уровни существуют).

1. Скопируй папку `Assets/LoopPuzzle/` из архива поверх своего проекта (файлы заменятся/добавятся).
2. Дождись компиляции без ошибок.
3. В меню Unity: **Tools → Loop Puzzle → Setup Iteration 3**

Скрипт добавит на сцену Game: HUD-панель сверху, попап победы (скрытый), объект LevelController, и прокинет все ссылки (включая связь PieceInput → LevelController). Сохранит сцену.

## 4. Как тестировать
1. Открой сцену `Assets/LoopPuzzle/Scenes/Game.unity`.
2. Нажми **Play**.
3. Сверху видно «Level 1» и «Moves: 0».
4. Тапай по углам — счётчик ходов растёт с анимацией.
5. Собери замкнутый квадрат (поверни все 4 угла правильно). Как только loop замкнулся — через ~0.4 сек всплывает попап «LEVEL COMPLETE» со звуком.
6. Нажми **RESTART** — уровень вернётся в исходное (скрамбленное) состояние, счётчик обнулится.
7. Нажми **NEXT** — пока перезагружает текущий уровень (переход на следующий будет в итерации 6).
8. Проверь Level_02: выбери GridManager → Current Level = Level_02 → Play.

## 5. Ожидаемый результат
- Счётчик ходов плавно увеличивается + лёгкий punch при каждом ходе.
- При сборе loop: звук победы, короткая пауза, затем затемнение экрана и попап с «отскоком» (Ease.OutBack).
- Во время победной анимации тапы по полю не срабатывают (ввод заблокирован).
- Кнопки попапа сжимаются при нажатии и звучат.
- Restart полностью сбрасывает уровень.

## 6. Известные ограничения текущей итерации
- Анимации «потока» по loop пока НЕТ (бегущая энергия по маршруту) — это итерация 4. Сейчас победа = звук + попап.
- Звёзды в попапе — серый плейсхолдер «★ ★ ★», без логики. Реальный расчёт — итерация 6.
- NEXT перезагружает текущий уровень (нет менеджера прогресса) — итерация 6.
- Поражения нет — в этой механике его и не предполагается (lose-условия появятся со спецмеханиками позже).
- Проверяется именно базовое условие: достаточно одного замкнутого loop, лишние неиспользованные элементы пока допустимы.

## 7. Что будет в следующей итерации
Анимация потока по замкнутому loop (подсветка + бегущая энергия + финальный burst), screen shake, squash & stretch — главный game feel победы.
